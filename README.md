# Escape Room MQTT Puzzle System

Система управления головоломками для квест-комнат. Контролирует четыре загадки (память, телефон, пятнашки, сейф), принимает MQTT-сообщения от Arduino через ESP8266, отображает статус и позволяет управлять состояниями через веб-интерфейс.

---

## 1. Для обычных пользователей (операторов)

### Быстрый запуск

1. Дважды кликните на файл **`start.bat`**
2. Дождитесь — откроется браузер с панелью управления

Всё, система работает.

### Как работает веб-интерфейс

На странице вы видите карточки загадок. Цвет карточки показывает состояние:

- **Красный** — устройство не в сети (не отвечает)
- **Жёлтый** — загадка загружена и готова к прохождению
- **Зелёный** — загадка решена, пройдена

**Клик по карточке** переключает состояние:
- Жёлтый -> клик -> Зелёный (имитация прохождения)
- Зелёный -> клик -> Жёлтый (сброс для повторного прохождения)
- Красный -> клик -> попытка активировать устройство

**Кнопки на панели:**
- **Обновить** — принудительно обновить данные с сервера
- **Git Update** — проверить наличие обновлений из репозитория

### Автообновление данных

Страница автоматически обновляется каждые 2 секунды — новые состояния от устройств появляются без перезагрузки.

### После прохождения

После того как загадка решена, она автоматически сбрасывается через 30 секунд, чтобы можно было пройти снова.

### Остановка системы

Закройте все появившиеся окна (Mosquitto, MQTT Listener, Web Server).

### Протокол MQTT

**Топики от загадок (Uno -> ESP -> Broker):**
- `puzzle/<name>` — состояние головоломки
- `/home/alive` — heartbeat устройства

**Форматы payload:**
- `STATE:ACTIVE` — загадка загружена, готова
- `STATE:COMPLETED` — загадка решена
- `UNLOCKED` — сигнал решения головоломки

**Топики команд (Broker -> ESP -> Uno):**
- `home/<name>` — команды управления
- `SET_STATE:ACTIVE` — активировать загадку
- `SET_STATE:COMPLETED` — установить как решённую

### Запуск для разработки

```bash
# Установка зависимостей
pip install -r requirements.txt

# 1. Запуск Mosquitto (первое окно)
"C:\Program Files\Mosquitto\mosquitto.exe" -v -c mosquitto.conf

# 2. Запуск MQTT-слушателя (второе окно)
cd server
python mqtt_listener.py

# 3. Запуск веб-сервера (третье окно)
cd server
python web_server.py
# -> http://localhost:5000

# 4. Тестовая отправка MQTT-сообщений (четвёртое окно)
cd tests
python test_sender.py
```

### Git-обновление при запуске

При старте `web_server.py` автоматически:
1. Определяет текущую ветку (`git rev-parse`)
2. Делает `git fetch origin <branch>`
3. Сравнивает HEAD с `origin/<branch>`
4. Если есть новые коммиты — делает `git pull`
5. После проверки запускает Flask-сервер

Если обновлений нет — просто продолжает запуск.

### База данных

SQLite-база `server/mqtt_messages.db` содержит три таблицы:
- `messages` — лог всех MQTT-сообщений
- `puzzle_states` — текущее состояние головоломок (active/completed)
- `puzzle_devices` — трекинг устройств (heartbeat, last_seen)

### REST API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/` | Главная страница (HTML) |
| `GET` | `/api/puzzles` | JSON список загадок с color/name/state |
| `POST` | `/api/puzzles/<name>/toggle` | Переключить состояние загадки |
| `POST` | `/api/puzzles/refresh` | Принудительное обновление (no-op, фронтовое) |
| `POST` | `/api/git/update` | Проверка и применение git-обновлений |

### Как добавить новую загадку

1. Прошейте новую Arduino и добавьте `.ino` в `firmware/<имя>/`
2. Добавьте имя в `KNOWN_PUZZLES` в `server/web_server.py`
3. Перезапустите `mqtt_listener.py` и `web_server.py`
